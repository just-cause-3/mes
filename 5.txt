XSS Detection Using OWASP ZAP in Kali Linux (Beginner Guide)
Step 1 — Open Terminal

Update packages:

sudo apt update
Step 2 — Install OWASP ZAP

Install using apt:

sudo apt install zaproxy -y

Check installation:

zaproxy

OR

owasp-zap

ZAP GUI should open.

Step 3 — Start OWASP ZAP

When ZAP opens:

Select:

No, I do not want to persist this session

Then click:

Start
Step 4 — Launch Browser Through ZAP

Inside ZAP:

Tools → Launch Browser

Choose Firefox.

This browser is already configured with ZAP proxy.

Step 5 — Open Vulnerable Website

Inside launched browser open:

testphp.vulnweb.com

Step 6 — Browse the Website

Click around:

Categories
Products
Search

This helps ZAP collect requests.

Step 7 — Use Search Box

Search for:

hello

Submit search.

Step 8 — Find Request in ZAP

In left panel:

Sites

Expand:

testphp.vulnweb.com

Find:

search.php

Click it.

Step 9 — Start Active Scan

Right-click:

search.php

Choose:

Attack → Active Scan

Click:

Start Scan
Step 10 — Wait for Scan

Bottom tab:

Active Scan

Wait until scan finishes.

Step 11 — Check Alerts

Go to:

Alerts

Look for:

Cross Site Scripting (Reflected)
Step 12 — Open Vulnerability Details

Double-click the alert.

You will see:

vulnerable parameter
payload used
request
response

Example payload:

<script>alert(1)</script>
Give feedback
Step 13 — Verify Manually

Copy vulnerable URL.

Open in browser.

If popup appears:

1

then XSS exists.

Manual Testing in Kali Browser

Try entering this in search box:

<script>alert(1)</script>
Give feedback

OR:

<img src=x onerror=alert(1)>
Give feedback

If popup appears → vulnerable.

Optional: Install Firefox in Kali

If missing:

sudo apt install firefox-esr -y
Optional: Start ZAP From Terminal
zaproxy

or

owasp-zap
Troubleshooting
Browser not opening from ZAP

Install Java:

sudo apt install default-jre -y

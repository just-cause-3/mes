# XSS Detection Using OWASP ZAP in Kali Linux
### Complete Beginner Guide — Anyone Can Follow This

---

## Before You Start — What Are We Doing?

**XSS (Cross-Site Scripting)** is a type of hacking technique where you trick a website into running your own code (a script) inside someone's browser.

**Example:** Imagine a search box on a website. Instead of searching for "shoes", you type in a piece of code. If the website is vulnerable, it runs that code — which could steal someone's data or show a fake popup.

**OWASP ZAP** is a free tool that acts like a "middleman" between your browser and a website. It watches all the traffic, and can automatically test the website for vulnerabilities like XSS.

**testphp.vulnweb.com** is a website that is *intentionally made vulnerable* for practice. It is 100% legal and safe to test on.

---

## What You Need
- Kali Linux running (on VirtualBox or directly)
- Internet connection
- A terminal window

---

---

## Step 1 — Open the Terminal

Look for the **black screen icon** on your Kali taskbar and click it.

OR press: `Ctrl + Alt + T`

A black window with a blinking cursor will appear. This is where you type commands.

> **How to run a command:** Type it exactly as shown, then press **Enter**.

---

## Step 2 — Update Your System

Before installing anything, update the package list:
```bash
sudo apt update
```

- It will ask for your **password** — type it and press Enter.
- You won't see the password as you type — that is normal, just keep typing.
- Wait for it to finish (you'll see the cursor return).

---

## Step 3 — Install OWASP ZAP

Now install ZAP:
```bash
sudo apt install zaproxy -y
```

- The `-y` means "yes, install automatically" — no need to confirm anything.
- This may take 1–3 minutes depending on your internet speed.
- Wait until you see the cursor return to normal.

---

## Step 4 — Install Firefox (if not already installed)

Check if Firefox is available:
```bash
firefox --version
```

If you see an error like `command not found`, install it:
```bash
sudo apt install firefox-esr -y
```

If it showed a version number, Firefox is already installed — skip this step.

---

## Step 5 — Fix "ZAP Won't Open" Issue (Java)

ZAP needs Java to run. Install it just in case:
```bash
sudo apt install default-jre -y
```

---

## Step 6 — Launch OWASP ZAP

Type this in the terminal:
```bash
zaproxy
```

Press Enter. Wait 10–20 seconds. A big colorful window will open — that is OWASP ZAP.

> If `zaproxy` doesn't work, try:
> ```bash
> owasp-zap
> ```

---

## Step 7 — Handle the First Popup

When ZAP opens, a small dialog box appears asking about saving your session.

Click:
```
No, I do not want to persist this session
```
Then click **Start**.

> This just means "don't save this session to a file." Fine for practice.

---

## Step 8 — Open a Browser Through ZAP

This is important — you must use ZAP's own browser, not your normal one.

Inside ZAP, click on the menu at the top:
```
Tools → Launch Browser → Firefox
```

A Firefox window will open. This browser is already set up to send all traffic through ZAP so ZAP can see and record everything.

---

## Step 9 — Go to the Practice Website

In the ZAP Firefox browser that just opened, click the address bar at the top and type:
```
http://testphp.vulnweb.com
```
Press **Enter**.

The website will load. It looks like a simple online art store. This is our target for testing.

---

## Step 10 — Browse Around the Website

Click on different parts of the website so ZAP can collect and map all the pages:

- Click **Categories** in the menu
- Click **Artists**
- Click **Disclaimer**
- Use the **Search** box — type anything like `hello` and press Search/Submit

> **Why?** Every time you click something, ZAP records that request. The more pages ZAP knows about, the better the scan.

---

## Step 11 — Find the Search Request in ZAP

Now go back to the ZAP window (don't close Firefox).

On the **left side** of ZAP, you will see a panel called **Sites**.

- Click the arrow/triangle next to `testphp.vulnweb.com` to expand it
- Keep expanding until you find a file called `search.php`
- Click on `search.php` once to select it

> `search.php` is the page that handles search — this is where we will test for XSS.

---

## Step 12 — Run the Active Scan

**Right-click** on `search.php` in the left panel.

A menu will appear. Click:
```
Attack → Active Scan
```

A new window opens. Just click **Start Scan** at the bottom.

> ZAP will now automatically inject many different test payloads (including XSS scripts) into the search box and check if the website runs them.

---

## Step 13 — Wait for the Scan

At the **bottom of ZAP**, click the tab that says **Active Scan**.

You will see a progress bar moving. Just wait — this usually takes **2–5 minutes**.

When the progress bar reaches 100% or stops moving, the scan is done.

---

## Step 14 — Check for XSS in Alerts

At the bottom of ZAP, click the **Alerts** tab (next to Active Scan).

Look through the list for:
```
Cross Site Scripting (Reflected)
```

If you see it — ZAP found an XSS vulnerability! ✅

> If you see it highlighted in **Red** or **Orange**, that means it is a high or medium severity finding.

---

## Step 15 — See the Details

**Double-click** on the `Cross Site Scripting (Reflected)` alert.

A details panel will open showing you:

| Field | What it means |
|---|---|
| **URL** | The exact page that is vulnerable |
| **Parameter** | Which input box was vulnerable (e.g., search box) |
| **Attack** | The payload ZAP used to find the vulnerability |
| **Evidence** | What the server returned that proved it was vulnerable |
| **Solution** | How the developer should fix it |

The payload ZAP used will look something like:
```html
<script>alert(1)</script>
```

---

## Step 16 — Verify Manually in the Browser

Copy the **URL** shown in the alert details.

Paste it into the ZAP Firefox browser address bar and press Enter.

If a popup box appears on the screen showing the number:
```
1
```
Then XSS is **confirmed and working**. The website ran your script!

---

## Step 17 — Try Manual XSS Payloads Yourself

Go back to the website's search box: `http://testphp.vulnweb.com`

Try typing these directly into the search box one at a time:

**Payload 1:**
```html
<script>alert(1)</script>
```
Click Search. If a popup with `1` appears → XSS found!

**Payload 2 (alternative):**
```html
<img src=x onerror=alert(1)>
```
Click Search. If a popup appears → XSS found!

> **What is happening?** These are tiny pieces of JavaScript code. If the website doesn't "clean" user input, it will run this code in the browser and show a popup — proving the vulnerability exists.

## Troubleshooting — Something Not Working?

**ZAP won't open / crashes**
```bash
sudo apt install default-jre -y
zaproxy
```

**Firefox doesn't open from ZAP**
```bash
sudo apt install firefox-esr -y
```
Then try Tools → Launch Browser → Firefox again.

**`zaproxy` command not found**
```bash
sudo apt install zaproxy -y
```

**No alerts found after scan**
- Make sure you browsed the website before scanning (Step 10)
- Make sure you searched for something (Step 10)
- Try right-clicking `testphp.vulnweb.com` (the whole site) instead of just `search.php` and run Active Scan on the entire site

**Scan is taking too long**
- Just wait — active scans can take up to 10 minutes on slower machines
- You can see progress in the Active Scan tab at the bottom

## Key Terms (Simple Definitions)

| Term | Meaning |
|---|---|
| **XSS** | Injecting scripts into a website to run in someone's browser |
| **Reflected XSS** | The injected script comes back immediately in the response (like a search result) |
| **Payload** | The test code we inject to check if a site is vulnerable |
| **Active Scan** | ZAP automatically attacks the site with many payloads to find vulnerabilities |
| **Alert** | A vulnerability that ZAP found and is reporting to you |
| **Proxy** | ZAP sits between your browser and the website, watching all traffic |

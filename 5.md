# XSS Detection Using OWASP ZAP in Kali Linux
### Beginner Step-by-Step Guide

> **Environment:** Kali Linux  
> **Target:** testphp.vulnweb.com (intentionally vulnerable practice site)  
> **Goal:** Detect Cross-Site Scripting (XSS) vulnerabilities using OWASP ZAP

---

## What is XSS?
Cross-Site Scripting (XSS) is a vulnerability where an attacker injects malicious scripts into a web page. When another user visits the page, the script executes in their browser. We use OWASP ZAP to automatically find these vulnerabilities.

---

## Step 1 — Update Packages

Open Terminal and run:
```bash
sudo apt update
```

---

## Step 2 — Install OWASP ZAP

```bash
sudo apt install zaproxy -y
```

Verify installation by launching it:
```bash
zaproxy
```
OR
```bash
owasp-zap
```
The ZAP GUI should open.

> If ZAP doesn't open, install Java first:
> ```bash
> sudo apt install default-jre -y
> ```
> Then try launching again.

---

## Step 3 — Start OWASP ZAP

When ZAP opens, a dialog will ask about session persistence.

Select:
```
No, I do not want to persist this session
```
Then click **Start**.

---

## Step 4 — Launch Browser Through ZAP

Inside ZAP, go to:
```
Tools → Launch Browser → Firefox
```

> This browser is **already configured** with ZAP's proxy — all traffic is automatically captured.

> If Firefox is missing:
> ```bash
> sudo apt install firefox-esr -y
> ```

---

## Step 5 — Open the Vulnerable Website

In the ZAP-launched Firefox browser, go to:
```
http://testphp.vulnweb.com
```

The site will load — it is an intentionally vulnerable practice website.

---

## Step 6 — Browse the Website

Click around the site to help ZAP collect and map requests:
- Click **Categories**
- Click **Products**
- Use the **Search** box

The more you browse, the more pages ZAP maps in the left panel.

---

## Step 7 — Use the Search Box

In the website's search box, type:
```
hello
```
Click Submit/Search.

This triggers a request to `search.php` which ZAP will capture.

---

## Step 8 — Find the Request in ZAP

In ZAP's left panel:
- Click **Sites**
- Expand `testphp.vulnweb.com`
- Find and click `search.php`

You will see the request and response in the right panel.

---

## Step 9 — Start Active Scan

Right-click on `search.php` in the Sites panel.

Select:
```
Attack → Active Scan
```

Click **Start Scan**.

> ZAP will now automatically inject various payloads into the search parameter to test for vulnerabilities.

---

## Step 10 — Wait for Scan to Finish

At the bottom of ZAP, click the **Active Scan** tab.

Watch the progress bar. Wait until the scan is complete (progress reaches 100%).

---

## Step 11 — Check Alerts

Click the **Alerts** tab at the bottom of ZAP.

Look for:
```
Cross Site Scripting (Reflected)
```

---

## Step 12 — Open Vulnerability Details

Double-click the **Cross Site Scripting (Reflected)** alert.

You will see a details window showing:
- **Vulnerable parameter** — which input field was exploited
- **Payload used** — the script ZAP injected (e.g., `<script>alert(1)</script>`)
- **Request** — the full HTTP request sent
- **Response** — what the server returned

---

## Step 13 — Verify Manually

Copy the vulnerable URL from the alert details.

Paste it in the browser — if an alert popup appears showing:
```
1
```
Then XSS is confirmed.

---

## Manual Testing

You can also test XSS manually in the search box:

**Payload 1 — Basic script tag:**
```html
<script>alert(1)</script>
```

**Payload 2 — Image tag (bypasses some filters):**
```html
<img src=x onerror=alert(1)>
```

If a popup appears after submitting either payload → the site is **vulnerable to XSS**.

---

## Troubleshooting

| Problem | Fix |
|---|---|
| ZAP won't open | Run `sudo apt install default-jre -y` then retry |
| Firefox missing | Run `sudo apt install firefox-esr -y` |
| Browser not opening from ZAP | Go to Tools → Options → Local Proxies, confirm port is 8080 |
| No alerts found | Make sure you browsed the site before scanning, repeat Step 6–9 |
| `zaproxy: command not found` | Run `sudo apt install zaproxy -y` again |

🔧 Step 1: Configure Java & Maven Inside Jenkins
Before creating any job, we need to tell Jenkins where Java and Maven are installed on your computer.

1.1 — Open Jenkins in Browser
http://localhost:8080
Login with your admin username and password.

1.2 — Go to Global Tool Configuration
On the Dashboard, click "Manage Jenkins" (left sidebar)
Click "Tools" (or "Global Tool Configuration" in older versions)
1.3 — Configure JDK
Scroll down to the JDK section:

Click "Add JDK"
Uncheck "Install automatically"
Fill in:
Field	Value
Name	JDK-21
JAVA_HOME	C:\Program Files\Java\jdk-21
⚠️ To find your exact Java path, run this in Command Prompt:

where java
It will show something like C:\Program Files\Java\jdk-21\bin\java.exe Remove \bin\java.exe — that is your JAVA_HOME path

1.4 — Configure Maven
Scroll down to the Maven section:

Click "Add Maven"
Uncheck "Install automatically"
Fill in:
Field	Value
Name	Maven-3
MAVEN_HOME	C:\Program Files\Maven\apache-maven-3.x.x
⚠️ To find your Maven path, run this in Command Prompt:

mvn -version
It will show the Maven home path like:

Maven home: C:\Program Files\Maven\apache-maven-3.9.x
Click "Save" at the bottom
📁 Step 2: Prepare Your Maven Project
We will use the exact project from Program 2. Make sure it exists at a known path on your PC.

Open Command Prompt and verify:

cd C:\Users\YourName\program2\myapp
dir
You should see pom.xml, src\ folder etc.

📝 Note down the full path of your project — you will need it in the next step. Example:

C:\Users\YourName\program2\myapp
🆕 Step 3: Create a New Jenkins Job
On Jenkins Dashboard, click "New Item" (top left)
You will see this screen:
┌──────────────────────────────────────────┐
│  Enter an item name:                     │
│  [MavenBuildJob________________]         │
│                                          │
│  ● Freestyle project                     │
│  ○ Pipeline                              │
│  ○ Multi-configuration project           │
│  ○ Folder                                │
└──────────────────────────────────────────┘
Type the name: MavenBuildJob
Select "Freestyle project"
Click OK
⚙️ Step 4: Configure the Job
You will now see the job configuration page. Follow each section:

4.1 — General Section
In the Description box, type:
Maven build and test job using Program 2 project
4.2 — Source Code Management
Select "None" (since we are using local files, not Git)
4.3 — Build Environment
Leave everything unchecked for now
4.4 — Build Steps (Most Important Part)
Scroll down to "Build Steps" section
Click "Add build step"
Select "Invoke top-level Maven targets"
A form appears — fill it in:

Field	Value
Maven Version	Maven-3 (the one we configured in Step 1)
Goals	clean install
Click "Advanced" button (small link below Goals)
In the POM field, enter the full path to your pom.xml:
C:\Users\YourName\program2\myapp\pom.xml
⚠️ Replace YourName with your actual Windows username

4.5 — Add a Second Build Step for Tests
Click "Add build step" again
Select "Invoke top-level Maven targets" again
Fill in:

Field	Value
Maven Version	Maven-3
Goals	test
Click Advanced → enter the same POM path again:

C:\Users\YourName\program2\myapp\pom.xml
4.6 — Post-Build Actions (See Test Results)
Scroll down to "Post-build Actions"
Click "Add post-build action"
Select "Publish JUnit test results report"
In the Test report XMLs field, enter:
C:\Users\YourName\program2\myapp\target\surefire-reports\*.xml
4.7 — Save the Job
Scroll to the bottom
Click "Save"
▶️ Step 5: Run the Build
You are now on the MavenBuildJob page.

Click "Build Now" (left sidebar)
You will see a new entry appear in "Build History" at the bottom left:
Build History
─────────────
#1  ● (running...)
The dot will turn:
🔵 Blue/Green = SUCCESS ✅
🔴 Red = FAILED ❌
📋 Step 6: View Build Output (Console Log)
Click on #1 in Build History
Click "Console Output" (left sidebar)
You will see the full build log. Look for these key lines:

[INFO] Scanning for projects...
[INFO] Building myapp 1.0-SNAPSHOT
[INFO] ------------------------
[INFO] BUILD SUCCESS
[INFO] ------------------------

-------------------------------------------------------
 T E S T S
-------------------------------------------------------
Running com.example.AppTest
Running test: 2 + 3 = 5
Tests run: 1, Failures: 0, Errors: 0, Skipped: 0

[INFO] BUILD SUCCESS
✅ This means Jenkins successfully compiled and tested your Maven project!

You will see a test report like:

🔄 Step 8: Integrate with Gradle (Same Project)
Now we will create a second job for the Gradle version using the same migrated project from Program 4.

8.1 — Configure Gradle in Jenkins Tools
Go to Manage Jenkins → Tools
Scroll to Gradle section
Click "Add Gradle"
Uncheck "Install automatically"
Fill in:
Field	Value
Name	Gradle-8
GRADLE_HOME	Your Gradle installation path
Find your Gradle path by running in CMD:

gradle -version
Look for Gradle home: line in the output

Click Save
8.2 — Create a New Gradle Job
Click "New Item" on Dashboard
Name it: GradleBuildJob
Select Freestyle project → Click OK
8.3 — Configure the Gradle Job
Scroll to Build Steps:

Click "Add build step"
Select "Invoke Gradle script"
Select "Use Gradle Wrapper" (if using gradlew) OR select "Invoke Gradle" and pick Gradle-8
Fill in:

Field	Value
Tasks	clean build test
Build File	Full path to your build.gradle
Example path:

C:\Users\YourName\program4\maven-example\build.gradle
Click Save → Click "Build Now"
📋 Step 9: View Gradle Build Console Output
Click #1 → Console Output. Look for:

> Task :compileJava
> Task :processResources
> Task :classes
> Task :compileTestJava
> Task :test

com.example.AppTest > testAdd PASSED

BUILD SUCCESSFUL in 3s
✅ Jenkins successfully ran your Gradle build and tests too!

✅ Final Summary
Job Name	Tool	Goals Run	Expected Result
MavenBuildJob	Maven	clean install + test	BUILD SUCCESS + Tests Passed
GradleBuildJob	Gradle	clean build test	BUILD SUCCESSFUL + Tests Passed
🆘 Common Errors & Fixes
Error	Fix
JAVA_HOME not set	Re-check JDK path in Manage Jenkins → Tools
Maven not found	Re-check Maven path in Manage Jenkins → Tools
pom.xml not found	Make sure the full path in POM field is correct
Build #1 shows red	Click Console Output — read the error message and share it
No test reports found	Run mvn test manually first to generate surefire-reports folder
Gradle wrapper not found	Use absolute path to gradlew.bat or install Gradle in Tools
💡 What you achieved: Jenkins is now automatically building and testing your Maven and Gradle projects with a single click — this is the foundation of CI/CD (Continuous Integration)!
